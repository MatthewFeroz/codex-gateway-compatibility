How compatible is “OpenAI-compatible”?

We ran the same Codex CLI, model, prompt, and environment through two model gateways. Connectivity was easy. Protocol fidelity decided the outcome.

OpenRouterPASS
Merge GatewayBLOCKED
Codex CLI 0.144.1 · GPT-5.6 Sol · real generation test

A gateway can accept authentication, resolve a model, and pass a health check—yet still fail as a coding-agent backend.

Agent clients exercise more of an API than a basic chat request: developer messages, tools, reasoning controls, streaming event order, usage accounting, and strict response schemas. We tested the whole path rather than stopping at “endpoint reachable.”

03 / FINDING

The gap is protocol conformance—not routing.

Merge Gateway’s OpenAI-compatible route accepted the request but returned a final streamed event that Codex 0.144.1 could not parse because a required input_tokens value was absent. Its native Responses route also rejected newer Codex request shapes, including additional_tools.

“OpenAI-compatible” should be validated with real clients, not only handcrafted HTTP requests.
Expected usage shape
{
  "usage": {
    "input_tokens": 123,
    "output_tokens": 45,
    "total_tokens": 168
  }
}
Why it matters

Codex treats response events as typed protocol objects. Missing required fields are not cosmetic; they terminate the agent session.

04 / WHAT WORKED ELSEWHERE

Merge works through other paths

Native SDK

Merge’s Python and TypeScript SDKs use its native Responses format and are the strongest route for custom agents.

Claude Code

Merge’s Anthropic-compatible endpoint completed a live Claude Code test with an Anthropic model.

LangGraph / Deep Agents

A custom agent can control request formatting and avoid strict Codex client assumptions, subject to tool-call testing.

06 / RECOMMENDATION

Build a client-level compatibility suite.

Test current and previous Codex versions against plain responses, streaming, developer messages, tool calls, reasoning controls, session continuation, and complete usage accounting. A gateway should only claim Codex compatibility after a real Codex process exits successfully.

Independent field test conducted on a Linux VPS. Results reflect the tested versions and endpoints on July 12, 2026 and may change as clients and gateways update.
